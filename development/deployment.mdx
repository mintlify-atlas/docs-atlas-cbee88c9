## Overview

pInk is optimized for deployment on Vercel with built-in configuration for serverless functions. The application uses Vite for the frontend build and Bun runtime for the backend API.

## Vercel Deployment

<Steps>
  <Step title="Prepare Your Repository">
    Ensure your code is pushed to a Git repository (GitHub, GitLab, or Bitbucket):

    ```bash
    git add .
    git commit -m "Prepare for deployment"
    git push origin main
    ```
  </Step>

  <Step title="Import Project to Vercel">
    1. Go to [vercel.com](https://vercel.com) and sign in
    2. Click **Add New** → **Project**
    3. Import your Git repository
    4. Vercel will auto-detect the configuration from `vercel.json`
  </Step>

  <Step title="Configure Environment Variables">
    In the Vercel project configuration, add your environment variables:

    | Variable | Value | Required |
    |----------|-------|----------|
    | `SUPABASE_URL` | `https://your-project.supabase.co` | Yes |
    | `SUPABASE_ANON_KEY` | Your Supabase anon key | Yes |
    | `NODE_ENV` | `production` | Recommended |

    See [Environment Setup](/development/environment) for detailed variable documentation.

    <Warning>
      Never commit your `.env` file to version control. Add it to `.gitignore`:
      
      ```bash
      echo ".env" >> .gitignore
      ```
    </Warning>
  </Step>

  <Step title="Deploy">
    Click **Deploy** and wait for the build to complete.

    Vercel will:
    1. Install dependencies using `bun install` (from `vercel.json:5`)
    2. Build the frontend with `bun run build` (from `vercel.json:6`)
    3. Configure serverless functions for the API
    4. Deploy to a production URL
  </Step>

  <Step title="Verify Deployment">
    Once deployed, test your endpoints:

    **Health Check**:
    ```bash
    curl https://your-app.vercel.app/health
    ```

    Expected response:
    ```json
    {
      "status": "OK",
      "timestamp": "2026-03-03T14:30:00.000Z",
      "version": "2.4.1"
    }
    ```

    **API Root**:
    ```bash
    curl https://your-app.vercel.app/api/comics
    ```
  </Step>
</Steps>

## Vercel Configuration

The `vercel.json` file configures the deployment:

```json
{
  "version": 2,
  "framework": "vite",
  "bunVersion": "1.x",
  "installCommand": "bun install",
  "buildCommand": "bun run build",
  "devCommand": "bun run dev",
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api/index.ts" },
    { "source": "/health", "destination": "/api/index.ts" },
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

**Key settings**:
- **framework**: Uses Vite build system
- **bunVersion**: Specifies Bun runtime version
- **rewrites**: Routes API requests to serverless function (`api/index.ts:1-3`)

## API Serverless Function

The API is deployed as a serverless function through `api/index.ts`:

```typescript
import { app } from "../src/server/index";

export default app;
```

This exports the Elysia app instance which Vercel converts to a serverless handler.

**Production behavior** (`src/server/index.ts:75-80`):

```typescript
if (process.env.NODE_ENV !== "production" && !process.env.VERCEL) {
  start().catch((err) => {
    console.error("❌ Failed to start server:", err);
    process.exit(1);
  });
}
```

In production/Vercel:
- Server does NOT auto-start on import
- Vercel handles request routing to the exported app
- Each request spawns a serverless function instance

## Available Scripts

From `package.json:6-11`:

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `vite` | Start Vite dev server (frontend only) |
| `server` | `bun run --watch src/server/index.ts` | Run API server with hot reload |
| `start` | `bun run src/server/index.ts` | Run API server (production mode) |
| `build` | `vite build` | Build frontend for production |
| `preview` | `vite preview` | Preview production build locally |

## Production Environment

### Environment Differences

**Development** (`NODE_ENV=development`):
- Swagger API documentation enabled (`src/server/index.ts:14`)
- Hot reload with `--watch` flag
- Detailed error logging
- CORS enabled for local development

**Production** (`NODE_ENV=production`):
- Swagger disabled (reduces bundle size)
- Optimized builds
- Error messages sanitized
- Serverless function execution

### Database Connection

The production database connection test (`src/server/config/database.ts:26-76`) runs on first API request:

```typescript
export async function testConnection() {
  console.log("🔍 Testing Supabase connection...");
  
  try {
    const { data, error, count } = await supabase
      .from("Comic")
      .select("*", { count: "exact" })
      .limit(1);
    
    if (error) {
      throw new Error(
        `Supabase query failed: ${error.message} (Code: ${error.code})`
      );
    }
    
    console.log("✅ Supabase connection SUCCESSFUL!");
    return { success: true, count, sampleData: data };
  } catch (error: any) {
    console.error("❌ Supabase connection FAILED!");
    throw error;
  }
}
```

## Monitoring and Logs

Vercel provides:
- **Real-time logs**: View function execution logs in Vercel dashboard
- **Analytics**: Track request volume and response times
- **Error tracking**: Automatic error logging

View logs:
1. Go to your Vercel project
2. Click **Deployments** → Select deployment
3. Click **Functions** → View logs

## Custom Domain

To add a custom domain:

1. In Vercel project settings, go to **Domains**
2. Add your domain (e.g., `comics.yourdomain.com`)
3. Configure DNS records as instructed
4. Vercel automatically provisions SSL certificate

## Rollback

To rollback to a previous deployment:

1. Go to **Deployments** in Vercel dashboard
2. Find the stable deployment
3. Click the three dots → **Promote to Production**
